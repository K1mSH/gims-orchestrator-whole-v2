#### 엔진 개선 1: 슬래시 operationId 지원

#### 엔진 개선 2: 응답 alias 대소문자 보존
사용자 발견: 응답 필드명이 전부 소문자.

원인: PG 가 unquoted identifier 를 자동 lowercase 로 정규화 (`AS REL_TRANS` → `AS rel_trans`).

수정: `DynamicQueryService.buildSelectClause()` 에서 alias 를 쌍따옴표로 감싸기 (`AS "REL_TRANS"`). 한 줄 변경.

검증: `curl /api/provide/megokrApi/ngw08?ctpv_nm=대전광역시` → `{"BRTC_NM":"대전광역시", ...}` 응답 확인 (이전 `brtc_nm` 소문자 → 지금 `BRTC_NM` 대문자).

- **target-meta-columns 일관성 이슈 종결**: provide 3종 명시는 "설계 의도의 명시적 표현"으로 인정, 자동 감지 도입 안함

- **레거시 URL 재현 우선**: operationId 에 슬래시 허용 하기 위해 엔진 수정. `feedback_no_regression_organic` 준수 (하위호환)
- **응답 필드 대소문자**: 레거시 소비자 호환을 위해 대문자 alias 필수 — PG 쌍따옴표 alias 로 해결

- **Service.update() immutable operationId**: 의도적 보호로 판단, DB 직접 UPDATE 로 우회. UI 수정 기능은 별도 이슈
