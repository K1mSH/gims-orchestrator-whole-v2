## 상세

### 설계 재검토 (계획서: `dev_plan/2026_04/02/saeol-snd-implementation.md`)

#### 레거시 분석 결과
- saeol.java: LINK_PLAN 1건씩 순회 → TABLE_NAME 16개 분기 → MERGE/DELETE
- TM_GD70001: 내부망 단일 row 테이블, 매 건 처리 후 LAST_CNTC_NO 갱신
- 비효율: 1건씩 MERGE, deduplicate 없음, 네트워크 분리, 2000줄 하드코딩

#### 확정 사항
1. **Step 방식**: 커스텀 SaeolLinkPlanSndStep + common 유틸 재사용 (SourceToIfStep 수정 안 함)
2. **커서 위치**: 같은 Oracle(29005)에 LINK_PLAN_CURSOR 테이블
3. **중복 처리**: Java에서 Map<테이블+PK, 최신row> deduplicate
4. **커서 갱신**: 전체 완료 후 1회 (MERGE idempotent)

### 구현

#### Oracle DDL (자동 생성)
- Oracle USER_TAB_COLUMNS 메타데이터 조회 → Python 스크립트로 IF_SND DDL 생성
- IF_SND: 소스 컬럼 + ID(IDENTITY) + SOURCE_REFS(UK) + LINK_STATUS + EXTRACTED_AT + UPDATED_AT + EXECUTION_ID
- LINK_PLAN_CURSOR: AGENT_CODE(PK), LAST_LINK_IDX, UPDATED_AT

#### 커스텀 Step 핵심 흐름
```
① LINK_PLAN_CURSOR → lastLinkIdx
② LINK_PLAN WHERE LINK_IDX > ? → 이벤트 조회
③ Java deduplicate (TABLE_NAME+PK → 최신 row, D flag skip)
④ TABLE_NAME별 그룹핑 → 배치 SELECT (YAML link-plan-keys로 WHERE 매핑)
⑤ Oracle USER_TAB_COLUMNS → MERGE SQL 동적 생성
⑥ 배치 MERGE INTO IF_SND (source_refs=SourceRefUtils, execution_id 설정)
⑦ SyncLog 저장 (Agent PG)
⑧ LINK_PLAN_CURSOR 갱신 (1회)
```

#### 이슈 해결
- Oracle connectionTestQuery: `SELECT 1` → `SELECT 1 FROM DUAL` (Others + Proxy)
- Orchestrator datasource 암호화: 평문 password → PasswordEncryptor 암호화 값
