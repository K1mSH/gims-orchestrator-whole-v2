### 오전 — DB 구조 전환 + 동적 SELECT 엔진

- api-provider DB를 Oracle(29004) → **PG 전용 컨테이너**(29006) 전환
  - 컨테이너: `gims_api_provider_pg` (postgres:15-alpine)
  - ojdbc 의존성 제거, PG 전용
- 제공용 엔티티 4개: Oracle 대문자 → **PG 소문자** 전환
- **ProviderDataSourceService**: Proxy(8093) 경유 접속정보 → JdbcTemplate 캐싱 (bojo-int 패턴)
- **DynamicQueryService**: 오퍼레이션 설정 → 동적 SQL 생성 + 실행
  - 연산자 10종: =, >, >=, <, <=, %포함%, 시작%, %끝, IN, BETWEEN
  - 보안: 테이블/컬럼명 정규식 화이트리스트 + PreparedStatement 바인딩
  - 페이징: PG LIMIT/OFFSET

### 오후 — 프론트 관리 UI 구현

- **목록 페이지** (`/api-provide`): 오퍼레이션 목록 + 활성/삭제
- **등록 페이지** (`/api-provide/new`):
  - Datasource select → 테이블 select (등록된 테이블만) → 컬럼 자동 로드
  - SELECT 컬럼 체크박스 + alias + 가공(ROUND/DATE_FORMAT/COALESCE/SUBSTRING)
  - WHERE 파라미터: 컬럼 드롭다운 추가 + 연산자/타입 자동 추론 + 필수/숨김
  - ORDER BY: 컬럼 드롭다운 + ASC/DESC
- **상세 페이지** (`/api-provide/[id]`): 6개 탭
  - 기본정보: 수정/저장
  - 컬럼: DB 컬럼 기반 체크박스 + alias + 가공
  - 파라미터: DB 컬럼 기반 WHERE 설정
  - 테스트: 동적 쿼리 실행 + SQL 하이라이팅 + 페이징
  - 명세서: API 호출 명세 자동 생성
  - 이력: 호출 이력 (페이징)

### 오후 — 외부 Gateway + API Key 검증

- **ApiGatewayController**: `GET /api/provide/{operationId}?apiKey=...&param=...`
  - 활성 오퍼레이션만 노출 (비활성=404)
  - API Key 검증 → 동적 쿼리 실행 → SQL 미포함 응답
- **MockApiKeyController**: 테스트용 키 검증 (하드코딩)
  - 운영 시 `app.api-key-validation.url`만 외부 API로 교체
- **ApiKeyValidationService**: 외부 검증 API 호출 래퍼
