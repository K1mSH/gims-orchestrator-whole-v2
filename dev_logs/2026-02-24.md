## 완료 작업: Skip→Failed 표시 버그 + 큰산 대문자 SOURCE 조회 실패 수정

### 이슈1: Skip이 Failed로 표시되는 문제
- **원인**: `SourceToIfStep.java` line 369-370에서 `skipCount`가 `saveSyncLogSummary`의 `failedCount` 파라미터 위치에 전달
- **수정**: `(long) writeCount, (long) skipCount, 0L` → `(long) writeCount, 0L, (long) skipCount`
- **결과**: 대전 obsvdata: `failed=0, skip=400` 정상 표시

### 이슈2: 큰산 SOURCE 테이블 상세조회 `column "id" does not exist` 에러
- **원인**: `ExecutionDataController.java`에서 PK 컬럼명을 `"id"` 하드코딩 → 큰산은 대문자 `"ID"` 사용
- **수정**: `findActualColumnName(sourceJdbc, actualTableName, "id")`로 대소문자 자동 해석
  - fallback: SyncLog의 `sourcePkColumn` (id 컬럼이 없는 테이블용)
- **결과**: 큰산 SEC_JEWON_VIEW 400건 정상 조회

### 수정 파일
1. `sync-agent-common/.../entity/SyncLog.java` - `sourcePkColumn` 필드 추가 (fallback용)
2. `sync-agent-common/.../step/SourceToIfStep.java` - skip/failed 파라미터 위치 수정, saveSyncLogSummary 오버로딩 추가
3. `sync-agent-common/.../controller/ExecutionDataController.java` - PK 하드코딩 → findActualColumnName 동적 해석

### 이슈3: totalCount에 skip 포함 → 건수 부자연스러움
- **원인**: `SyncLog.getTotalCount()` = success + failed + skip → IF obsvdata가 `건수=400, 성공=0, 실패=0`으로 표시
- **수정**: `getTotalCount()` = success + failed (skip 제외), summary API도 동일 수정
- **결과**: IF obsvdata: `건수=0, 성공=0, 실패=0` (skip은 별도 필드에만 보존)

### 이슈4: SOURCE와 IF 건수 불일치
- **원인**: SOURCE SyncLog에 `readCount`를 successCount로 저장 → IF가 0인데 SOURCE가 400으로 표시
- **수정**: SOURCE도 `writeCount`를 successCount로 저장, skipCount도 IF와 동일하게 반영
- **결과**: SOURCE/IF 건수 일치 (jewon: 400/400, obsvdata: 0/0)
