### 2. SourceToIfStep id 컬럼 제외 처리
- **문제**: 외부 DB의 `id`가 IF 테이블의 `id`(auto-increment PK)를 덮어써서 다중 소스 간 PK 충돌
  - daejeon id=1,2,3... 과 infoworld id=1,2,3...이 같은 IF 테이블에서 충돌
- **해결**: SELECT/INSERT 컬럼 분리
  - `selectColumns`: id 포함 (SELECT용, source_refs PK 추출에 필요)
  - `columns` (insertColumns): id 제외 (INSERT용, IF 테이블 자체 auto-increment 사용)
- **수정 파일**: `sync-agent-common/.../step/SourceToIfStep.java`
  - `AUTO_INCREMENT_PK_COLUMNS = List.of("id")` 상수 추가
  - `resolveColumns()` 후 SELECT용/INSERT용 분리
- **결과**: source_refs가 `["E:8:26:1"]` (id 기반)으로 정상 생성

### 3. trace-source 정밀 매칭 수정
- **문제**: Loader trace-source에서 `source_refs LIKE '%4905%'`가 다른 datasource의 동일 PK도 매칭 (2건 반환)
  - `["E:8:27:4905"]` (daejeon)과 `["E:9:28:4905"]` (infoworld) 모두 매칭
- **해결**: 정확 매칭 우선 → LIKE fallback
  - 1차: `source_refs = '["E:8:27:4905"]'` (정확 매칭)
  - 2차: LIKE fallback (복합 source_refs 등 예외용)
- **수정 파일**: `sync-agent-common/.../controller/ExecutionDataController.java`

### 4. Datasource fallback 순서 최적화
- **문제**: trace-source/source 엔드포인트에서 TARGET 먼저 fallback → dev DB에 빈 테이블 존재 → 0건 반환
- **해결**: SOURCE(외부 DB) 먼저 fallback → TARGET 순서로 변경
  - trace-source: 소스 데이터 조회이므로 SOURCE 우선이 맞음
  - source: 동일하게 SOURCE 우선
- **수정 파일**: `sync-agent-common/.../controller/ExecutionDataController.java` (2곳)
