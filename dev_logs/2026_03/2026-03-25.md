### 1. API Collector 테스트 호출 URL 인코딩 수정

- **증상**: 테스트 호출 시 `Invalid character '지' for QUERY_PARAM` 에러
- **원인**: 3/24 이중인코딩 수정에서 `build(true)` 사용 → 한글 미인코딩 → `URI.create()` 에러
- **최종 해결**: `URLEncoder.encode()` + `build(true)` 조합
  - 한글(`지하수`) → `%EC%A7%80...` 정상 인코딩
  - API키(`qaj/1Mkn...`) → `qaj%2F1Mkn...` 정상 인코딩
  - `+`, `=` 등 특수문자 모두 정상
- **추가 수정**: Mock API 키 데이터를 인코딩된 형태(`%2F`) → 원본 Base64(`/`)로 변경
- **수정 파일**: `ApiCallService.java`, `MockApiController.java`

### 9. 커스텀 실행기 구조 설계 + 안양 이용량 1차 구현

#### 설계 논의 과정
- **문제**: 안양시 이용량 API는 2개 API 호출 + 건별 서브쿼리 산출 + JOIN 후처리 → 범용 매핑으로 불가
- **검토한 방안**:
  - A안: api-collector에 커스텀 실행기 추가 → **채택**
  - B안: agent에 API 소스 타입 추가 → 모니터링 체계가 달라서 보류
- **커스텀을 api-collector에 넣는 이유**: 범용만 가능하면 커스텀 나올 때마다 agent로 가야 하고, api-collector 존재 이유가 약해짐
- **등록 UI 논의**: 커스텀은 API URL/매핑이 불필요 → "일반/프리셋" 유형 분리로 결정
- **원본 분석**: main args로 날짜 받는 단발성 배치 → 우리는 스케줄 자동화 + 어제~오늘 범위로 전환

#### 구현 내용
- **백엔드**:
  - `CustomExecutor` 인터페이스 + `CustomExecutionResult` + `CustomExecutorRegistry`
  - `ApiEndpoint.executorType` 필드 추가
  - `ApiExecutionService.run()` 커스텀/범용 분기
  - `AnyangUsageExecutor` 구현 (FAC UPSERT + DATA UPSERT + LEGACY INSERT)
  - `AnyangApiFac`, `AnyangApiData`(복합PK), `UseLegacyData` 엔티티
  - Mock API 2개 (`/mock/anyang/fac`, `/mock/anyang/data`)
  - `CreateRequest.url` @NotBlank 제거 (프리셋은 URL 불필요)
- **프론트엔드**:
  - 등록 폼 최상단 "일반/프리셋" 라디오 선택
  - 프리셋: 실행기 + API명 + Target Datasource만 입력
  - 상세 화면: 커스텀 배지, 매핑 탭 비활성화, 수동 실행 가능
- **엔티티 수정**:
  - `AnyangApiData`: id 제거 → account_no + meter_dtm 복합 PK
  - DATA INSERT → ON CONFLICT 복합키 UPSERT 추가
  - `UseLegacyData.last_change_dt` → NOW() (SYSDATE)

### 10. Portainer Unhealthy + Eureka 미등록 트러블슈팅

- **증상**: 신규 서비스 컨테이너가 Portainer에서 unhealthy, Eureka 대시보드에 미등록
- **원인 분석**:
  - Docker HEALTHCHECK가 `/app/env/port-{서비스명}.env` 파일로 포트를 읽는데, 파일 미존재
  - Eureka 등록 로그가 한 줄도 없음 → 클라이언트 초기화 자체가 안 됨
  - 기존 서비스 대비 `spring-cloud-starter-config` 의존성 누락 확인
- **유력 원인**: Config Server를 안 쓰더라도 `spring-cloud-starter-config` 의존성이 Spring Cloud Context 초기화에 영향 → Eureka 클라이언트 미초기화
- **다음 액션**: config 의존성 추가 후 재배포 테스트
