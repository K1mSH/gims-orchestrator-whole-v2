# 2026-03-13 작업 일지

## 작업 내용: infolink-api-collector 모듈 개발 (Step 1~4)

### 신규 모듈: infolink-api-collector
- 외부 API 데이터를 등록/수집/적재하는 독립 모듈 (port 8084)
- 코드 변경 없이 UI에서 API 등록만으로 수집 가능한 구조

### Step 1: Entity + CRUD + UI
- 엔티티: ApiEndpoint, ApiParam, ApiFieldMapping, ApiSchedule, ApiExecutionHistory
- DTO: ApiEndpointDto, ApiScheduleDto, ApiExecutionHistoryDto, TestCallDto
- Controller: ApiEndpointController, ApiScheduleController, ApiHistoryController, MetadataController
- Service: ApiEndpointService, ApiScheduleService
- 프론트: 목록/상세 페이지, InfoTab (기본정보 + 파라미터 CRUD)

### Step 2: Test Call + Tree UI
- DynamicParamResolver: STATIC/DYNAMIC(TODAY/NOW) 파라미터 해석
- ApiCallService: RestTemplate 기반 호출 (PATH/QUERY/BODY/HEADER, BASIC/BEARER 인증)
- ResponseParser: JSON → TreeNode 트리 변환, dataRootPath 기반 레코드 추출
- ApiTestService: 테스트 호출 오케스트레이션
- 프론트 MappingTab: 테스트 호출 → JSON 트리 뷰 → 데이터 루트 선택

### Step 3: Field Mapping + Execution
- DataTransformer: 값 변환 (DATE_FORMAT, NUMBER)
- ApiExecutionService: 범용 실행 엔진 (API 호출 → 파싱 → 매핑 → INSERT/UPSERT)
- MetadataController: 자체 DB 스키마 조회 (테이블/컬럼 목록)
- 프론트: 타겟 테이블 선택, 필드 매핑 테이블, 실행 버튼, HistoryTab

#### 버그 수정
- `$` 경로 파싱: `navigateTo()`에서 `$` = root로 처리 추가
- PG 트랜잭션 aborted: Savepoint 패턴으로 행 단위 에러 격리
- `@Transactional` + 별도 DataSource Connection 조합으로 JPA 세션 + 독립 커밋 양립

#### E2E 테스트 결과
| 시나리오 | 결과 |
|---------|------|
| 최초 INSERT 100건 | insertCount=100, skipCount=0 |
| 중복 INSERT (upsert OFF) | insertCount=0, skipCount=100 |
| 중복 UPSERT (upsert ON) | insertCount=100, skipCount=0 |

### Step 4: Schedule
- ApiScheduleExecutor: TaskScheduler + CronTrigger 기반 스케줄 실행
- SchedulerConfig: ThreadPoolTaskScheduler (4스레드)
- ApiScheduleService: CRUD 시 executor register/unregister 연동
- 프론트 ScheduleTab: cron 프리셋, 생성/수정/토글/삭제
- LazyInitializationException 해결: `@Transactional` 재적용
- 스케줄 자동 실행 확인: triggeredBy=SCHEDULE로 이력 기록
