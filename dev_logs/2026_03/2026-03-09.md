# 2026-03-09 작업 일지

## SyncLog 테이블-매핑 리디자인 + E2E 파이프라인 검증

### 작업 목적
SyncLog를 per-table(tableName/tableType) 방식에서 per-mapping(mappingName/sourceTables/targetTables) 방식으로 전환.
N:M 테이블 매핑을 명시적으로 추적하고, Link 테이블을 SyncLog/모니터링에서 제외.

### 주요 변경사항

- **SyncLog.java**: 엔티티 리디자인

### 발견된 이슈 (미해결 → 내일 작업)
1. **Loader/SND Source(IF) 추적 불가**
   - Loader의 source = IF 테이블, IF의 execution_id는 RCV것
   - Loader execution_id로 IF 데이터 조회 시 0건
   - 해결안: IF가 source인 경우 target의 source_refs 기반으로 IF 조회
   - 계획 문서: `dev_plan/2026-03-09/loader-source-trace-fix.md`

2. **tm_gd970101이 Target으로 표시됨**
   - Internal Loader SyncLog/YAML에서 target에 포함되어 있음
   - link 테이블 성격이므로 target에서 제거 필요
