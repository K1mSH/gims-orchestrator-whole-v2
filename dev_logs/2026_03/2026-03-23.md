# 2026-03-23 작업 일지

## 작업 내용

### 1. api-collector LOOKUP 파생 컬럼 기능

**목적**: 외부 API 응답에 없는 컬럼을 내부 공통코드 기반으로 파생 생성
- 예: `orignl_url` → 도메인 추출 → GIMS 공통코드(NGW_0118) → 언론사명

**설계**:
- 기존 `ApiFieldMapping` 엔티티에 LOOKUP 필드 통합 (별도 엔티티 없이)
- `isDerived` 플래그로 1:1 매핑과 파생 컬럼 구분
- 정규식(`extractPattern` + `extractGroup`)으로 키 추출 — 프리셋 제공 (도메인/숫자/괄호)
- LOOKUP API URL은 `application.yml`의 `lookup.common-code-url` 설정, UI에서는 그룹코드만 입력
- `lookupMatchType` 필드 예비 (향후 CONTAINS 등 확장)

**구현**:
- `ApiFieldMapping` 엔티티: LOOKUP 전용 필드 9개 추가
- `TransformType` 확장: SUBSTRING, TRIM, REPLACE, DEFAULT_VALUE, LOOKUP
- `DataTransformer`: 단순 변환 4종 구현
- `LookupService` 신규: 내부 API 호출 → Map 캐싱 → 정규식 키 추출 → LOOKUP
- `ApiExecutionService`: 일반/파생 매핑 분리 처리, LOOKUP Map 사전 로딩
- `CreateRequest`에 적재 설정 필드 추가 (등록 시 한번에 저장)

### 4. 버그 수정

- **정규식 이중 이스케이핑**: 프리셋 `www\\\\.` → `www\\.`으로 수정 (Java Pattern에서 매칭 실패 원인)
- **등록 플로우 불완전 저장**: create → update 2단계를 create 한번에 통합 (중간 실패 시 데이터 유실 방지)
- **프리셋 버튼 값 미반영**: React state 배치 업데이트 문제 — 연속 setState를 한번에 처리
- **상세 페이지 hooks 에러**: useState가 early return 뒤에 위치 → 위로 이동
