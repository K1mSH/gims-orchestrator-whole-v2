### 오전

#### 1. 서비스명 매핑 문서 작성
- `docs/useIncludeJeju/service-name-mapping.md` 작성 ✔
- 레거시 12개 프로그램 ↔ 신규 서비스명 1:1 매핑
- DMZ 5개 서비스 (API Collector 커스텀 실행기) 이름 확정
- Internal 6개 서비스 (bojo-int 커스텀 Loader) 이름 미정

#### 2. 아키텍처 방향 결정
- 제주 Internal은 **bojo-int 파이프라인 편입 안 함** → 별도 커스텀 Step (C안: 패키지 분리) ✔
- bojo-int에 `jeju/step/`, `saeol/step/` 패키지로 분리
- IF 테이블: **테이블별 개별 IF 쌍** (팀장님 합의) ✔
- 전체 파이프라인 흐름 확정: API Collector → DMZ DB → bojo-dmz SND → Proxy → bojo-int RCV → 커스텀 Loader → GIMS

#### 3. DB 환경 정비
- DMZ 새올 DB 환경: Tibero 도커 라이선스 제한 → **Oracle XE로 대체** ✔
  - 컨테이너: `gims_dmz_saeol_oracle` (29005), 계정: k1m/1111
- 새올 16개 테이블 **전부 실존** 확인 (이전 5개 → 16개 정정) ✔

#### 4. DMZ DB 환경
- 보조망 DB: 제주/이용량/나라장터/뉴스/안양 등 전부 포함
- 새올 DB: Tibero (실서버) / Oracle (개발) — 우리는 읽기만
