### 오전 수정 사항

#### 1. HikariCP connectionTestQuery Oracle 호환 (bojo-int, proxy-internal)
- `SELECT 1` → Oracle은 `SELECT 1 FROM DUAL` 분기
- 수정: SyncDataSourceService.java (bojo-int), ProxyDataSourceService.java (proxy-internal)

#### 2. InternalBojoLoadStep write=0 시 FAILED 처리
- read>0 & write=0일 때 기존 SUCCESS → FAILED + 에러메시지 반환
- PipelineRunner는 기존 FAILED 처리 그대로 활용

#### 3. ExecutionDataController Oracle 호환 (common)
- `LIMIT` → `pagingSql()`/`limit1Sql()` 헬퍼로 Oracle OFFSET/FETCH 대응 (9곳)
- `information_schema.tables` → Oracle `user_tables` 분기
- `SELECT COUNT(*) ... LIMIT 1` → LIMIT 제거 (COUNT는 항상 1행)

#### 4. Source 추적 대량 PK 처리 (common)
- 같은 DB: 서브쿼리 방식 (`WHERE id IN (SELECT ... FROM target WHERE execution_id = ?)`)
- 다른 DB(cross-DB): 1,000건 단위 배치 분할 실행 + 결과 합산
- sameDb 판단: Agent 기본값이 아닌 Execution 레코드의 실제 datasourceId 비교

#### 5. Proxy Internal에 ojdbc11 추가
- 추적 조회 시 Proxy가 Oracle에 직접 연결 → 드라이버 필요

#### 6. 프론트 Oracle 대문자 컬럼 대응
- `row.source_refs` → `row.SOURCE_REFS` 추가 (행 클릭 추적)

#### 7. test_plan 추적 검증 기준 강화
- 기존: 건수 확인만
- 변경: 건수 + 단건 역추적 + 프론트 UI 검증 3단계 필수
