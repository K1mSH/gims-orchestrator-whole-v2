- 대시보드/에이전트 관리에서 상태 확인이 주기적으로 필요

  - `@Scheduled(fixedDelay=30000, initialDelay=10000)`
  - `CompletableFuture`로 전체 Agent 병렬 health check
  - RUNNING 상태 제외 (실행 흐름에서 관리)
  - 상태 변경 시에만 DB write + 로그

## 내부망 Loader (Internal Loader) 구현

### 배경
내부 서비스망에서 IF_RSV 관측데이터를 최종 GIMS 서비스 DB로 적재하는 Loader가 필요.
레거시(Oracle iBatis) → PostgreSQL + JDBC 기반 리메이크. EAV 확장(1→3행) 포함.

### 핵심 설계 포인트
- **제원(tm_gd970001)은 GIMS 서비스가 자체 관리하는 마스터 데이터**
  - 외부 업체 제원은 IF_RSV에서 끊김 (target으로 안 감)
  - Loader는 제원을 READ ONLY로 사용 (spot_id 매핑용)
- **관측데이터만 적재** (IF_RSV → pm_gd970201)
- DMZ Loader 패턴 동일: 기본 UPSERT, 시간설정도 UPSERT (DELETE 없음)

### 데이터 흐름
```
tm_gd970001 (GIMS 마스터)  ← Loader READ (spot_id 매핑)
```
