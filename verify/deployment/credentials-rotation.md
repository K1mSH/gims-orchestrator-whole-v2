# Credentials Rotation — 계정 / API 키 교체

실배포 진입 시 **반드시 교체되어야 할 모든 자격증명** 목록 + 교체 절차.

## 1. 원칙
- dev 계정 (`k1m` / `1111`) 은 실배포에 **절대 잔존 금지**.
- credentials 는 **yml / 코드에 하드코딩 금지** — 환경변수 / secret manager 만 허용.
- 교체 후 **git 히스토리에 남은 구 값 삭제 불필요** — 다만 회전(rotation) 이력으로 기록.
- **정기 회전** 일정 수립 (실배포 후에도 분기별 등).

## 3. 교체 절차 (표준)

1. 신규 값 발급 / 생성
2. 환경변수 / secret 에 신규 값 주입
3. 서비스 재기동 / 무중단 리로드 (가능한 경우)
4. 동작 확인 (해당 서비스 health + 기능 스모크)
5. 구 값 폐기 (가능한 시점에 외부 시스템 등록에서도 삭제)
6. 본 문서 § 4 회전 이력 기록
