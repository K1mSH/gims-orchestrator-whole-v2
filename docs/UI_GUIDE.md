# 2. 대시보드

**경로:** `/`

## 2.1 제공 기능

| 기능 | 사용자 가치 |
|------|-------------|
| 통계 카드 | 전체 현황을 한눈에 파악 (Agent 수, 실행 현황, 오류 현황) |
| Agent 상태 테이블 | 각 Agent의 온라인/오프라인 상태 및 마지막 실행 결과 확인 |
| 최근 실행 이력 | 최근 실행된 작업 목록과 결과 확인 |
| 자동 갱신 | 10초마다 자동으로 데이터 갱신 (수동 새로고침 불필요) |

## 2.2 통계 카드

| 항목 | 설명 |
|------|------|
| 전체 Agent | 등록된 Agent 총 개수 |
| 온라인 Agent | 현재 정상 연결된 Agent 수 |
| 오프라인 Agent | 연결 안 되는 Agent 수 (문제 확인 필요) |
| 현재 실행 중 | 지금 동기화 진행 중인 Agent 수 |
| 오늘 실행 | 오늘 실행된 총 횟수 |
| 오늘 실패 | 오늘 실패한 횟수 (빨간색 표시) |

## 2.3 Agent 상태 테이블

| 컬럼 | 설명 |
|------|------|
| Agent | Agent 이름 (클릭 시 상세 페이지 이동) |
| Zone | 소속 Zone |
| 상태 | ONLINE / OFFLINE / RUNNING |
| 마지막 실행 결과 | SUCCESS / FAILED / RUNNING |
| 마지막 실행 시간 | 마지막 실행 시작 시각 |

## 2.4 최근 실행 이력 테이블

| 컬럼 | 설명 |
|------|------|
| Agent | Agent 이름 (클릭 시 실행 상세 페이지 이동) |
| 타입 | RCV / Loader / SND |
| 상태 | SUCCESS / FAILED / RUNNING |
| 읽기/쓰기/스킵 | 처리된 데이터 건수 |
| 소요시간 | 실행에 걸린 시간 |
| 트리거 | MANUAL / SCHEDULE / API |
| 시작 시간 | 실행 시작 시각 |

**Datasource & 테이블 설정:**

Agent가 연결하는 Source/Target DB와 처리 대상 테이블 정보를 확인하고 관리

| 기능 | 사용자 가치 |
|------|-------------|
| Datasource 확인 | Agent가 연결하는 Source/Target DB 정보 확인 |
| 테이블 목록 확인 | 동기화 대상 테이블 목록 확인 |
| 컬럼 목록 확인 | 각 테이블의 컬럼 목록 확인 |
| 테이블 선택/해제 | 수정 모드에서 처리 대상 테이블 변경 |

**표시 정보:**

```
┌─────────────────────────────────────────────────────────┐
│ Datasource & 테이블                                     │
├───────────────────────────┬─────────────────────────────┤
│ Source                    │ Target                      │
├───────────────────────────┼─────────────────────────────┤
│ dmz_ent1_ds (PostgreSQL)  │ dmz_target_ds (PostgreSQL)  │
│ 192.168.1.10:5432         │ 192.168.1.20:5432           │
├───────────────────────────┼─────────────────────────────┤
│ SEC_JEWON                 │ sec_jewon                   │
│ → id, obsv_code, obsv_name│ → id, obsv_code, obsv_name  │
│   sido, sigungu, x, y ... │   sido, sigungu, x, y ...   │
├───────────────────────────┼─────────────────────────────┤
│ SEC_OBSVDATA              │ sec_obsvdata                │
│ → obsv_code, obsv_date,   │ → obsv_code, obsv_date,     │
│   obsv_time, gwdep ...    │   obsv_time, gwdep ...      │
└───────────────────────────┴─────────────────────────────┘
```

**사용 시나리오:**
| 상황 | 활용 방법 |
|------|-----------|
| 테이블 구조 확인 | 동기화 대상 테이블의 컬럼 목록 확인 |
| 데이터 매핑 검토 | Source와 Target의 컬럼 구조 비교 |
| 테이블 추가 | 수정 모드에서 새로운 테이블 선택 |
| 테이블 제외 | 특정 테이블을 동기화 대상에서 제외 |

---

**스케줄 관리:**

| 기능 | 사용자 가치 |
|------|-------------|
| 스케줄 추가 | Cron 표현식으로 자동 실행 주기 설정 |
| 스케줄 수정 | 실행 주기 변경 |
| 스케줄 활성화/비활성화 | 스케줄 삭제 없이 일시 중지 |
| 스케줄 삭제 | 자동 실행 제거 |

**Cron 표현식 예시:**
| 표현식 | 의미 |
|--------|------|
| `0 0 * * * *` | 매시 정각 |
| `0 0/30 * * * *` | 30분마다 |
| `0 0 9 * * *` | 매일 오전 9시 |
| `0 0 9,18 * * *` | 매일 오전 9시, 오후 6시 |
| `0 0 0 * * MON` | 매주 월요일 자정 |

**사용 시나리오:**
- 업무 시간에만 동기화: `0 0 9-18 * * MON-FRI` (평일 9시~18시 매시)
- 야간 배치: `0 0 2 * * *` (매일 새벽 2시)
- 테스트 기간 일시 중지: 스케줄 비활성화 → 테스트 완료 후 재활성화

---

**실행 버튼:**

| 버튼 | 설명 |
|------|------|
| 상태 확인 | Agent 서버 연결 상태 확인 (Health Check) |
| 즉시 실행 | 기본 설정으로 동기화 실행 |
| 시간 지정 실행 | 특정 기간 데이터 재동기화 |

**상태 확인 (Health Check):**
- Agent 서버가 정상 동작하는지 확인
- 네트워크 문제, 서버 다운 등 감지
- 결과: ONLINE / OFFLINE

**즉시 실행:**
- 스케줄과 관계없이 지금 바로 동기화 실행
- 증분 동기화 (이미 처리된 데이터는 건너뜀)
- 사용 시나리오: 긴급하게 데이터 동기화가 필요할 때

**시간 지정 실행:**
- 특정 기간의 데이터를 강제로 재동기화
- 시작 시간 / 종료 시간 입력 가능
- 주의: 해당 기간 데이터가 전체 재처리됨

**시간 지정 실행 사용 시나리오:**
| 상황 | 사용 방법 |
|------|-----------|
| 특정 날짜 데이터 오류 | 해당 날짜만 시작/종료 시간 지정하여 재동기화 |
| 과거 데이터 누락 발견 | 누락된 기간 지정하여 재수집 |
| 외부 시스템 데이터 수정됨 | 수정된 기간 지정하여 다시 가져오기 |

### 4.2.3 모니터링 탭 (Monitor)

| 기능 | 사용자 가치 |
|------|-------------|
| 실행 상태 | 현재 실행 중인 작업의 진행 상황 |
| Source 데이터 | 이번 실행에서 읽어온 Source 데이터 목록 |
| Target 데이터 | 이번 실행에서 적재한 Target 데이터 목록 |
| 실시간 갱신 | 실행 중일 때 자동으로 데이터 갱신 |

---

**실행 상태 정보:**
| 항목 | 설명 |
|------|------|
| 상태 | RUNNING / SUCCESS / FAILED |
| 진행률 | 처리 완료 비율 (%) |
| 읽기 건수 | Source에서 읽어온 데이터 수 |
| 쓰기 건수 | Target에 적재한 데이터 수 |
| 스킵 건수 | 처리하지 않고 건너뛴 데이터 수 |
| 소요 시간 | 현재까지 걸린 시간 |

---

**Source 데이터 조회:**

현재 실행에서 Source에서 읽어온 데이터 목록을 테이블로 표시

| 컬럼 | 설명 |
|------|------|
| ID | 데이터 고유 ID |
| 관측소 코드 | obsv_code |
| 상태 | PENDING / SUCCESS / FAILED |
| 추출 시간 | 데이터가 추출된 시각 |
| source_refs | 원본 출처 정보 (클릭 시 상세 확인) |

**사용 시나리오:**
- 데이터가 제대로 읽어졌는지 확인
- 특정 관측소 데이터가 포함되었는지 검색
- 읽기 오류 발생 시 어떤 데이터에서 문제인지 확인

---

**Target 데이터 조회:**

현재 실행에서 Target에 적재된 데이터 목록을 테이블로 표시

| 컬럼 | 설명 |
|------|------|
| ID | 데이터 고유 ID |
| 관측소 코드 | obsv_code |
| 상태 | PENDING / SYNCED |
| execution_id | 이 데이터를 적재한 실행 ID |
| source_refs | 원본 출처 정보 |

**사용 시나리오:**
- 데이터가 Target까지 정상 도달했는지 확인
- 특정 데이터의 처리 상태 확인
- 원본 출처 역추적 (source_refs 클릭)

---

**데이터 검색 및 필터링:**

데이터 목록에서 검색/필터 기능으로 원하는 데이터를 빠르게 찾기

| 기능 | 설명 |
|------|------|
| 키워드 검색 | 입력한 키워드로 데이터 검색 |
| 컬럼 지정 검색 | 특정 컬럼만 대상으로 검색 |
| 상태 필터 | PENDING / SUCCESS / FAILED 상태로 필터링 |
| 정렬 | 원하는 컬럼 기준으로 오름차순/내림차순 정렬 |

**제원 데이터 검색 가능 컬럼:**
| 컬럼 | 설명 | 검색 예시 |
|------|------|----------|
| id | 데이터 ID | `123` |
| obsvCode | 관측소 코드 | `GPM-1234` |
| obsvName | 관측소명 | `서울` |
| sido | 시도 | `경기도` |
| sigungu | 시군구 | `성남시` |
| sourceRefs | 원본 출처 | `dmz_ent1` |

**관측데이터 검색 가능 컬럼:**
| 컬럼 | 설명 | 검색 예시 |
|------|------|----------|
| id | 데이터 ID | `456` |
| obsvCode | 관측소 코드 | `GPM-1234` |
| remark | 비고 | `점검` |
| sourceRefs | 원본 출처 | `dmz_ent1` |

**검색 사용 시나리오:**
| 상황 | 검색 방법 |
|------|-----------|
| 특정 관측소 데이터 찾기 | obsvCode 컬럼에서 관측소 코드 검색 |
| 특정 지역 데이터 찾기 | sido 또는 sigungu 컬럼에서 지역명 검색 |
| 실패한 데이터만 보기 | 상태 필터에서 FAILED 선택 |
| 특정 Source에서 온 데이터 | sourceRefs 컬럼에서 데이터소스 ID 검색 |

---

**데이터 추적 (Trace):**

특정 데이터 row를 클릭하면 해당 데이터의 전체 흐름을 추적

```
[Source] sec_jewon (GPM-1234-5678-00001)
    ↓
[IF_RSV] if_rsv_sec_jewon (id: 123, status: SUCCESS)
    ↓
[Target] sec_jewon (id: 456, status: PENDING)
```

**사용 시나리오:**
- 데이터가 어디서 멈췄는지 확인
- 오류 발생 시 어느 단계에서 문제인지 파악
- 원본 데이터와 Target 데이터 비교

### 4.2.4 히스토리 탭 (History)

| 기능 | 사용자 가치 |
|------|-------------|
| 실행 이력 목록 | 과거 실행 기록 (성공/실패/소요시간) |
| 실행 상세 조회 | 특정 실행의 상세 정보 (처리 건수, 오류 내역) |
| 데이터 추적 (Trace) | 특정 데이터가 어디서 왔는지 역추적 |

---

**실행 이력 목록:**

| 컬럼 | 설명 |
|------|------|
| 실행 ID | 고유 식별자 (클릭 시 상세 페이지 이동) |
| 상태 | SUCCESS / FAILED / RUNNING |
| 읽기/쓰기/스킵 | 처리 건수 요약 |
| 소요 시간 | 전체 실행 시간 |
| 트리거 | MANUAL(수동) / SCHEDULE(스케줄) / API(외부호출) |
| 시작 시간 | 실행 시작 시각 |
| 종료 시간 | 실행 종료 시각 |

---

**트리거 유형:**

| 트리거 | 설명 | 발생 상황 |
|--------|------|-----------|
| MANUAL | 사용자가 UI에서 직접 실행 | "즉시 실행" 또는 "시간 지정 실행" 버튼 클릭 |
| SCHEDULE | 스케줄에 의한 자동 실행 | Cron 표현식에 따라 자동 트리거 |
| API | 외부 시스템에서 API 호출 | 다른 시스템이 REST API로 실행 요청 |

---

**실행 상세 정보:**

실행 ID를 클릭하면 해당 실행의 상세 정보 페이지로 이동

| 섹션 | 내용 |
|------|------|
| 기본 정보 | 실행 ID, Agent, 상태, 시작/종료 시간, 소요시간 |
| 처리 통계 | 읽기/쓰기/스킵 건수, 성공/실패 비율 |
| Step별 결과 | 각 처리 단계의 개별 결과 |
| 오류 내역 | 실패 시 오류 메시지 및 스택 트레이스 |
| 처리 데이터 | 이 실행에서 처리된 데이터 목록 |

---

**오류 분석 시나리오:**

1. **실행 실패 확인**: 히스토리에서 FAILED 상태인 실행 찾기
2. **오류 내역 확인**: 실행 상세에서 오류 메시지 확인
3. **문제 데이터 특정**: 어떤 데이터에서 오류가 발생했는지 확인
4. **원인 파악**: source_refs로 원본 데이터 확인
5. **재처리**: 문제 해결 후 해당 기간 시간 지정 실행

---

**실행 비교:**

여러 실행의 처리 건수를 비교하여 이상 징후 파악

| 확인 사항 | 의미 |
|-----------|------|
| 평소보다 읽기 건수가 적음 | Source 데이터 누락 또는 조회 조건 문제 |
| 평소보다 스킵 건수가 많음 | 데이터 품질 문제 또는 중복 데이터 |
| 소요 시간이 급격히 증가 | DB 성능 문제 또는 대량 데이터 처리 |
| 특정 시간대에 실패 집중 | 네트워크 또는 외부 시스템 문제 |

# 5. Agent 체인

**경로:** `/chains`

## 5.1 제공 기능

| 기능 | 사용자 가치 |
|------|-------------|
| 체인 구성 | 여러 Agent를 순차적으로 실행하도록 연결 |
| 체인 실행 | 체인 단위로 한 번에 실행 |
| 실행 순서 관리 | RCV → Loader → SND 순서 등 정의 |

## 5.2 사용 예시

```
체인: "전체 동기화"
  1. RCV (외부 → IF_RSV)
  2. Loader (IF_RSV → Target)
  3. SND (Target → IF_SND)
```

**장점:**
- 클릭 한 번으로 전체 파이프라인 실행
- 각 단계 완료 후 다음 단계 자동 실행
- 중간 실패 시 이후 단계 중단
