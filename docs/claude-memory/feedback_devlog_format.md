dev_logs 작성 시 **요약 + 상세** 구조.

**Why:** 비개발자 관리자에게 매일 오전/오후/내일할일을 손으로 적어 제출. 요약을 바로 옮겨쓸 수 있어야 한다.

**How to apply:**
- `## 요약 (보고용)`: 오전/오후/내일할일 **각 1~2줄**로 짧게. 세부 나열 금지. 프론트 수정 5건이면 "프론트 UI 개선 (재수집 모달, 정렬, 설명 등)" 한 줄로 묶기
- 진행도 테이블은 유지 (상태 변경된 것만 강조)
- `## 상세`: 수정 파일, 원인/해결, 테스트 결과 등 기술적 내용
- 요약이 길어지면 반드시 줄이기 — 매번 지적받고 있음
