단위테스트 결과서는 원본(NGIS1_B02_단위테스트결과서v1.8.xlsx) 양식을 정확히 따라야 함.

**Why:** 최종적으로 원본 엑셀에 복붙해서 합칠 산출물이라 컬럼 구조가 다르면 안 됨.

**How to apply:**
- 컬럼 구조: A~U 21컬럼 고정 (Test ID, 화면명, 화면ID, 프로그램ID, 관련테이블, 테스트주요내용(F), 입력값(G), 예상결과값(H), 테스트결과(개발) I~K, 테스트결과(PL) L~P, 시정조치 Q~S, 최종완료일자(T), 비고(U))
- F열(테스트 주요 내용): 원본에서 숨김 처리되어 있으나 데이터는 존재 — `○ 기능 설명` 형태로 작성
- G열(입력값): 화면 진입 경로 안내문구 먼저 쓰고 (`사이드바 > 메뉴 클릭`) 이후 ① ② ③ 단계별 동작
- 컬럼을 임의로 추가/제거하지 않기 — 원본에 없는 컬럼 만들면 복붙 불가
- 병합 헤더: F3:H3=사양서, I3:K3=테스트결과(개발), L3:P3=테스트결과(PL), Q3:S3=시정조치
- 스타일: 맑은 고딕 10pt, 헤더 fill=B4C6E7, A~E center, F~H left
- 시나리오 분류: 화면(UI_010_XX) 단위로 나누되, 화면 안에서는 기능 흐름으로 묶기 (버튼 하나하나 쪼개지 않기)
- 파일 위치: `v2_docs/scenario/` (개발 외적 산출물이라 orchestrator_v2 밖)
- md 먼저 내용 검수 → 확인 후 xlsx 생성 순서
