목록 vs 상세 페이지의 액션 배치 룰:

- **상세 페이지(따로 있는 라우트)가 있는 도메인** — 삭제 등 위험 액션은 **상세 안**에. 목록에는 두지 않음 (예: api-collect, api-provide, agents)
- **상세가 모달인 도메인** — 액션을 목록 행에 둬도 됨 (예: datasources)

**Why:**
- 목록에서 행 클릭 = 상세 진입 → 사용자가 의도와 다르게 삭제 버튼 클릭 위험
- 같은 도메인 내 액션 위치 일관성 (상세 페이지에 모든 변경 액션 묶음)
- 목록은 조회/필터 흐름 위주, 상세는 변경 흐름 위주로 역할 분리

**How to apply:**
- 새 목록 페이지 KRDS화 시 삭제/비활성/수정 같은 위험 액션이 행에 있다면 → 상세 페이지로 이동 검토
- 사용자가 명시적으로 "목록에 두자" 요청 시 예외 (드물게)
- 모달 상세인 도메인은 그대로 유지
