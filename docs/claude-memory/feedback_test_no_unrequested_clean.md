테스트 중 사용자가 "추가로 X 검증해봐 / where 실행 다시 해봐 / 추적현황 확인해봐" 라고 하면, 그건 **기존 데이터·상태 위에 추가 실행만** 하라는 뜻. 임의로 DB 클린(TRUNCATE)/link_ngwis 재설정/재초기화 하지 말 것.

**Why:** 이미 클린테스트(§4 E2E 등)를 끝낸 상태에서 또 클린하면 (1) 사용자가 어느 게 어느 테스트 결과인지 추적 불가, (2) 끝낸 검증을 다시 해야 하는 작업 중복, (3) 사용자가 보려던 데이터가 날아감. 2026-05-11 03-bojo 테스트 중 "where 실행 다시 해봐"를 클린 재실행으로 오해해서 §4 E2E 데이터를 TRUNCATE → 사용자 짜증.

**How to apply:**
- "클린테스트 해 / 처음부터 다시" 라고 **명시적으로** 말할 때만 TRUNCATE/재초기화
- "추가로 ~ 확인 / ~ 다시 해봐 / 건수 비교해봐" = 기존 상태 그대로 두고 그 항목만 추가 실행
- 데이터가 어수선해서 클린이 필요해 보이면 → 클린하기 전에 사용자에게 먼저 물어볼 것
- 별개: execution_history 누적이 거슬린다고 하면 그건 이력 테이블만 TRUNCATE (데이터 테이블 아님)
