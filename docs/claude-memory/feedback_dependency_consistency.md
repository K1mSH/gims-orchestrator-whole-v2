우리 시스템 모듈끼리는 같은 의존성(GAV/버전)을 써야 한다. 모듈마다 다른 버전이 들어가는 건 잘못된 상태로 본다.

**Why**: 2026-05-06 폐쇄망 Nexus 호환 점검 중 ojdbc8/ojdbc11 혼재 발견 — 사용자가 "우리 시스템끼리는 같은 의존성 써야 한다, jdbc 버전 다른 건 말이 안 된다"고 명시. 폐쇄망 Nexus 미러링 부담뿐 아니라 운영 일관성/디버깅 용이성/CVE 추적 단일화 측면에서 모듈 간 차이를 허용하지 않는 정책.

**How to apply**:
1. 의존성 점검 작업 시 모든 모듈을 횡단 비교 — 같은 라이브러리에 다른 GAV/버전이 있는지 확인.
2. 차이 발견 시 통일 방향 제안 (보통 더 보편적/안정적/사내망 보유 버전으로).
3. 새 의존성 추가 시 다른 모듈에서 같은 라이브러리를 쓰는지 확인하고 동일 GAV/버전 채택.
4. Spring Boot BOM 같은 BOM 기반 관리는 자동 일관성 확보 — 가능한 BOM 따라가기.
5. java-library 모듈(BOM 미사용, 예: infolink-agent-common)이라도 명시 버전은 BOM 친화 버전과 일치시킬 것.
