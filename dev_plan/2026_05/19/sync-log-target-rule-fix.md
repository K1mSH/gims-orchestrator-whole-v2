-- 새올 sync_log (target = saeol-oracle 일관 룰)
-- 다른 DB sync_log entity (SyncLog.java) 의 컬럼 정의와 일치
CREATE TABLE "K1M"."SYNC_LOG" (
    "ID"                NUMBER GENERATED BY DEFAULT AS IDENTITY NOT NULL ENABLE,
    "EXECUTION_ID"      VARCHAR2(100) NOT NULL,
    "STEP_ID"           VARCHAR2(50),
    "MAPPING_NAME"      VARCHAR2(100),
    "SOURCE_TABLES"     VARCHAR2(4000),

### Phase 2 — `SyncLogWriter` 시그너처 정정

**현 시그너처** (호출처 12):
```java
public static void save(SyncLogRepository repo, String executionId, ...)
```

**신 시그너처**:
```java
public static void save(DataSourceProvider dsp, String executionId, ...)
```

**내부 `persist()` 변경**:
```java
private static void persist(DataSourceProvider dsp, String executionId, String stepId, ...) {
    String targetDs = dsp.getTargetDatasourceId();
    JdbcTemplate jdbc = dsp.getJdbcTemplate(targetDs);
    try {
        jdbc.update(
            "INSERT INTO sync_log (execution_id, step_id, mapping_name, " +
            "source_tables, target_tables, source_pk_column, " +
            "read_count, write_count, failed_count, skip_count, " +
            "failed_keys, error_summary, created_at) " +
            "VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)",
            executionId, stepId, mappingName, sourceJson, targetJson, sourcePkColumn,
            readCount, writeCount, failedCount, skipCount,
            failedKeys != null ? String.join(",", failedKeys) : null,
            errorSummary, new Timestamp(System.currentTimeMillis())
        );
    } catch (Exception e) {
        log.warn("[{}] SyncLog 저장 실패 (매핑: {}): {}", stepId, mappingName, e.getMessage());
    }
}
```

> SQL 컬럼 양식은 PG/Oracle 공용 (소문자 컬럼). Oracle 의 `CREATED_AT DEFAULT SYSTIMESTAMP` 가 NULL 시 동작하므로 INSERT 에서 `created_at` 생략 가능 — 하지만 PG `sync_log` 는 DEFAULT 없이 NOT NULL 일 수 있어 명시 박는 게 안전.

> `repo.save(entry)` → JDBC INSERT 직접. JpaRepository 의존성 제거.
