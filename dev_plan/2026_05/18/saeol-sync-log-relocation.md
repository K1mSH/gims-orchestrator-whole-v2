### Phase 2 — agent 코드: SyncLogWriter 동적 datasource

현재 `SyncLogRepository extends JpaRepository<SyncLog, Long>` — JPA primary 만 사용.

**옵션 A — 새올 전용 별 EntityManager**:
- others-dmz 모듈에 새올용 secondary EntityManager 추가 (saeol-oracle)
- `SyncLogRepository` 분기 또는 새올 전용 별 Repository
- 다른 agent 코드 무변경
- 단점: 새올만 예외 코드, 일관성 깨짐

**옵션 B — SyncLogWriter 동적 JdbcTemplate (권장)**:
- `SyncLogWriter.save(...)` 가 `dataSourceProvider.getJdbcTemplate(targetDatasourceId)` 사용
- JpaRepository 우회 — JdbcTemplate 으로 직접 INSERT
- 모든 agent 일관 (target 의 sync_log 에 박음)
- 35 agent: target = module default 라 결과 동일 (실 변경 영향 X)
- 새올: target=saeol-oracle → saeol-oracle 에 INSERT (Phase 1 DDL 필요)

**권장 = B**.

코드 변경:
```java
// SyncLogWriter (common)
public static void save(... DataSourceProvider dataSourceProvider, String targetDatasourceId ...) {
    JdbcTemplate jdbc = dataSourceProvider.getJdbcTemplate(targetDatasourceId);
    jdbc.update("INSERT INTO sync_log (...) VALUES (...)", ...);
}
```

각 Step (DefaultLoadStep, SaeolLinkPlanSndStep 등) 의 SyncLogWriter.save 호출처에 targetDatasourceId 전달.

### Phase 3 — backend `resolveManagementDatasource` 단순화
