# 클래스명세서 생성 가이드라인

## 1. 개요

orchestrator_v2 프로젝트의 실제 소스코드를 기반으로 NGIS D23 양식의 클래스명세서(docx)를 모듈별로 생성한다.
Python(python-docx)으로 docx를 자동 생성하며, 소스 변경 시 스크립트 수정 후 재생성한다.

## 3. 대상 클래스 타입 (6종)

| 타입 | 클래스 타입 표기 | 테이블 행 수 | 오퍼레이션 |
|------|-----------------|-------------|-----------|
| Controller | Class <<Controller>> | 9행 | 있음 (어노테이션 + 시그니처) |
| Service | Class <<Service>> | 9행 | 있음 (시그니처만) |
| Request | Class <<Request>> | 7행 | 없음 |
| Response | Class <<Response>> | 7행 | 없음 |
| Entity | Class <<Entity>> | 7행 | 없음 |
| Repository (interface) | Interface <<Repository>> | 9행 | 있음 (커스텀 메서드) |
| Repository (class) | Class <<Repository>> | 9행 | 있음 |

**제외 대상**: Enum, Config, Util, Application, Step, Pipeline 등

## 4. DTO → Request/Response 변환 규칙

- DTO 클래스 자체는 명세서에 작성하지 **않음**
- DTO 내부 `*Request` inner class → **하나의 Request 클래스로 합집합**
- DTO 내부 `*Response` inner class → **하나의 Response 클래스로 합집합**
- 클래스명: `{DTO접두사}Request`, `{DTO접두사}Response`
  - 예: `AgentDto` → `AgentRequest`, `AgentResponse`

## 5. 문서 구조

### 5.1 도메인 그룹핑

Controller의 도메인 기준으로 그룹을 나누고, 그룹 내에서 클래스 타입별로 정렬한다.

```
1    {도메인명}
1.1    Controller
1.1.1    {ControllerName}
1.2    Service
1.2.1    {ServiceName}
1.3    Request
1.3.1    {RequestName}
1.4    Response
1.4.1    {ResponseName}
1.5    Entity
1.5.1    {EntityName1}
1.5.2    {EntityName2}
1.6    Repository
1.6.1    {RepositoryName}
2    {다음 도메인명}
...
```

- 해당 타입이 없으면 섹션 생략 (번호 자동 조정)
- Entity/Repository가 여러 개면 `.1`, `.2`로 이어감

### 5.2 빈칸 처리

- **클래스ID**: 빈칸
- **화면ID**: 빈칸
- **인터페이스번호**: 없음 (소스 기반이므로)

## 6. 각 클래스 타입별 작성 규칙

### 6.1 Controller

**클래스 개요**:
```
개요
-    {ClassName}을 정의한다.

상세내용
-    {제공 기능 나열}
```

**속성**: `private final {Service} {serviceName};`

**오퍼레이션**: 실제 소스의 어노테이션 + 시그니처 그대로
```java
// {한글 설명}
@GetMapping("/{id}")
public ResponseEntity<AgentDto.Response> getAgent(@PathVariable Long id);
```

### 6.2 Service

**속성**: 주입된 의존성 전체 나열

**오퍼레이션**: 어노테이션 없이 주석 + 시그니처만
```java
// {한글 설명}
public AgentDto.Response findById(Long id);
```

### 6.3 Request / Response

**속성**: 필드별 한 줄씩 (합집합)
```java
private String agentCode;    // 에이전트 고유 코드
private Long id;             // PK
```

**오퍼레이션**: 없음 (7행 테이블)

### 6.4 Entity

**속성**: JPA 어노테이션을 별도 줄에, 필드 블록 사이 빈 줄
```java
@Id @GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;                     // PK

@Column(name = "agent_code", unique = true, nullable = false)
private String agentCode;            // 에이전트 고유 코드
```

**오퍼레이션**: 없음 (7행 테이블)

**클래스 개요**: 테이블명, 역할, 연관관계 설명 포함

### 6.5 Repository (JPA interface)

**속성**: `extends JpaRepository<{Entity}, {IdType}>`

**오퍼레이션**: 커스텀 메서드만 (기본 CRUD 제외). `@Query`/`@EntityGraph` 포함.
```java
// 에이전트 코드로 조회
Optional<Agent> findByAgentCode(String agentCode);
```

### 6.6 Repository (JdbcTemplate class)

**속성**: 주입된 의존성

**오퍼레이션**: public 메서드 나열

## 9. 수정/재생성 방법

### 소스 변경 시

1. 해당 모듈의 스크립트(`scripts/generate_*.py`)에서 데이터 수정
   - 메서드 추가/삭제 → operations_text 수정
   - 필드 추가/삭제 → attributes_text 수정
   - 새 클래스 추가 → 그룹 데이터에 클래스 추가
2. 스크립트 실행: `python scripts/generate_*.py`
3. output 폴더에 docx 재생성 (같은 파일명 덮어쓰기)

### 버전 올릴 때

스크립트 내 output 파일명의 버전을 `v0.1` → `v0.2` 등으로 변경

### 패키지 경로 일괄 변경

```bash
cd scripts/
sed -i 's/old\.package\./new.package./g' generate_*.py
# 전체 재생성
for f in generate_orchestrator.py generate_api_collector.py generate_bojo.py generate_bojo_int.py generate_proxy_dmz.py generate_proxy_internal.py; do python "$f"; done
```

## 10. 스크립트 공통 구조

모든 생성 스크립트는 동일한 패턴:

```python
# 1. 공통 헬퍼 함수 (docx 서식)
#    set_cell_shading, _make_run, set_cell_text, set_cell_multiline,
#    merge_cells_horizontal, set_table_borders, set_table_layout_fixed,
#    set_table_indent, set_row_height, set_column_widths,
#    set_cell_border_nil, create_class_table

# 2. 데이터 정의 (모듈별 클래스 정보)
#    - package_name, class_name, class_type
#    - class_overview (개요 + 상세내용)
#    - attributes_text (속성)
#    - operations_text (오퍼레이션, 있는 경우)

# 3. 문서 생성 (Heading + 테이블)
#    - doc.add_heading() 으로 번호 체계
#    - create_class_table() 으로 명세서 테이블

# 4. 저장
#    - doc.save(output_path)
```

`create_class_table()` 함수 시그니처:
```python
def create_class_table(doc, package_name, class_id, class_name, screen_id,
                       class_type, class_overview, attributes_text,
                       operations_text=None):
```
- `class_id`, `screen_id`: 항상 `''`
- `operations_text=None`: 7행 테이블 (Request/Response/Entity)
- `operations_text` 제공: 9행 테이블 (Controller/Service/Repository)
