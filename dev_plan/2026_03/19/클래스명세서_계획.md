# Orchestrator v2 클래스명세서 작업 계획

## 1. 작업 개요

**목적**: orchestrator_v2 프로젝트의 실제 소스코드를 기반으로 모듈별 클래스명세서(NGIS_D23 양식)를 생성한다.

**대상 모듈** (6개):

| # | 모듈명 | 패키지 루트 | 역할 |
|---|--------|------------|------|
| 1 | sync-agent-bojo | com.sync.agent.bojo | DMZ 통합 Agent (RCV/Loader/SND) |
| 2 | sync-agent-bojo-int | com.sync.agent.bojoint | Internal 통합 Agent (RCV/Loader) |
| 3 | sync-orchestrator/backend | com.sync.orchestrator | 중앙 오케스트레이터 (Spring Boot) |
| 4 | sync-proxy-dmz | com.sync.proxy.dmz | DMZ DB 프록시 |
| 5 | sync-proxy-internal | com.sync.proxy.internal | Internal DB 프록시 |
| 6 | infolink-api-collector | com.infolink.collector | 외부 API 수집 모듈 |

> **제외**: sync-agent-common (공통 라이브러리 — 별도 문서 불필요)

## 2. 작성 규칙

### 2.1 대상 클래스 타입

| 타입 | 클래스 타입 표기 | 포함 조건 |
|------|-----------------|----------|
| Controller | Class <<Controller>> | `@RestController` / `@Controller` 클래스 |
| Service | Class <<Service>> | `@Service` / 비즈니스 로직 클래스 |
| Request | Class <<Request>> | DTO 내부 `*Request` inner class 또는 독립 Request 클래스 |
| Response | Class <<Response>> | DTO 내부 `*Response` inner class 또는 독립 Response 클래스 |
| Entity | Class <<Entity>> | `@Entity` JPA 엔티티 클래스 |
| Repository | Class <<Repository>> | `JpaRepository` 확장 인터페이스 |

**제외 대상**: Enum, Config, Util, Application, Step, Pipeline 등

### 2.2 DTO 처리 규칙

- DTO 클래스 자체를 명세서에 작성하지 **않음**
- DTO 내부에 `*Request`, `*Response` inner class가 있으면 → **각각 별도 클래스**로 작성
- inner class가 여러 개인 경우 (예: `CreateRequest`, `UpdateRequest`) → **하나의 Request 클래스로 합쳐서** 속성 합집합 작성
- 클래스명: `{DTO명}Request`, `{DTO명}Response` (예: `AgentDto` → `AgentRequest`, `AgentResponse`)

### 2.3 Entity 작성 규칙

- 속성(Attributes): JPA 필드 전체 나열 (`@Id`, `@Column`, 연관관계 등)
- 오퍼레이션(Operations): **없음** (7행 테이블 — 오퍼레이션 Row 생략)
- 테이블명 표기: 클래스 개요에 `@Table(name = "xxx")` 정보 포함

```
개요
-    Agent 엔티티를 정의한다.

상세내용
-    테이블: agent. Agent 정보를 관리하는 JPA 엔티티로, 코드/이름/URL/상태/실행이력 등을 포함한다.
```

### 2.4 Repository 작성 규칙

- 속성(Attributes): 없음 (빈칸)
- 오퍼레이션(Operations): 커스텀 쿼리 메서드만 나열 (JpaRepository 기본 메서드 제외)
- 클래스 타입: `Interface <<Repository>>`

```java
// Agent 코드로 조회
Optional<Agent> findByAgentCode(String agentCode);
```

### 2.5 클래스ID / 인터페이스번호

- 둘 다 **비워둠** (빈칸)

### 2.6 Controller 오퍼레이션 규칙

소스코드의 실제 메서드 시그니처를 기반으로 작성:

```java
// {메서드 설명 (한글)}
@GetMapping("/{id}")
public ResponseEntity<AgentDto.Response> getAgent(@PathVariable Long id);
```

- 실제 소스의 어노테이션, 파라미터, 반환타입을 **그대로** 사용
- `throws Exception` 은 실제 소스에 있는 경우만 포함
- 주석은 메서드 역할을 한글로 작성

### 2.7 Service 오퍼레이션 규칙

```java
// {메서드 설명 (한글)}
public AgentDto.Response findById(Long id);
```

- 어노테이션 없이 주석 + 시그니처만
- 실제 소스의 반환타입/파라미터 그대로 사용

### 2.8 클래스 개요 작성

```
개요
-    {클래스명}을(를) 정의한다.

상세내용
-    {제공 기능 나열}
```

### 2.9 속성(Attributes) 작성

- **Controller**: `private final {Service} {serviceName};` (실제 주입된 의존성)
- **Service**: `private final {Repository} {repositoryName};` (실제 주입된 의존성)
- **Request/Response**: 필드별 한 줄씩
  ```
  private String agentCode;    // Agent 코드
  private Long id;             // ID
  ```
- **Entity**: JPA 필드 전체
  ```
  @Id @GeneratedValue
  private Long id;                    // PK
  @Column(unique = true)
  private String agentCode;           // Agent 코드
  private String agentName;           // Agent 명
  @Enumerated(EnumType.STRING)
  private AgentStatus status;         // 상태
  @OneToMany(mappedBy = "agent")
  private List<Schedule> schedules;   // 스케줄 목록
  ```
- **Repository**: 없음 (빈칸)

## 5. 문서 구조

### 번호 체계 (모듈 내)

도메인 그룹 단위로 넘버링. Entity/Repository도 그룹 내에 포함:

```
1    Agent
1.1    Controller
1.1.1    AgentController
         [클래스 명세서 테이블]
1.2    Service
1.2.1    AgentService
         [클래스 명세서 테이블]
1.3    Request
1.3.1    AgentRequest
         [클래스 명세서 테이블]
1.4    Response
1.4.1    AgentResponse
         [클래스 명세서 테이블]
1.5    Entity
1.5.1    Agent
         [클래스 명세서 테이블]
1.5.2    AgentTable
         [클래스 명세서 테이블]
1.6    Repository
1.6.1    AgentRepository
         [클래스 명세서 테이블]
2    Callback
2.1    Controller
...
```

- 그룹핑: Controller의 도메인 기준 (agent, callback, chain, datasource, execution, schedule)
- 해당 타입이 없는 그룹: 해당 섹션 생략

### Heading 스타일

- **Heading 1**: `{번호}    {그룹명}` (예: "1    Agent")
- **Heading 2**: `{번호.번호}    {클래스타입}` (예: "1.1    Controller")
- **Heading 3**: `{번호.번호.번호}    {클래스명}` (예: "1.1.1    AgentController")
