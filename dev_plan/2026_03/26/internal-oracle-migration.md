# Internal Agent DB: PostgreSQL → Oracle 전환

## 목적
- bojo-int의 **IF + Target DB 전체**를 PG(29001/dev) → **Oracle(29004)**로 전환
- 실운영 내부망 환경(Oracle/Tibero)과 동일한 개발 환경 구축
- Agent 자체 DB(SyncLog, JPA 엔티티)는 PG(29001/dev) 유지

## 전환 범위

| 구분 | 현재 (PG 29001/dev) | 전환 후 (Oracle 29004/XEPDB1) |
|------|---------------------|-------------------------------|
| IF_RSV 테이블 (Internal RCV 쓰기) | PG | **Oracle** |
| GIMS Target 테이블 (Internal Loader 쓰기) | PG | **Oracle** |
| Agent 자체 DB (SyncLog, JPA) | PG | PG 유지 |

### Oracle 대상 테이블 (6개)

| 테이블 | 용도 | 접근 Agent |
|--------|------|-----------|
| IF_RSV_SEC_JEWON | IF 수신 제원 | Internal RCV (write), Internal Loader (read) |
| IF_RSV_SEC_OBSVDATA | IF 수신 관측데이터 | Internal RCV (write), Internal Loader (read) |
| TM_GD970001 | 관측소 제원 | Internal Loader (read) |
| PM_GD970201 | 관측자료 (EAV) | Internal Loader (write) |
| TM_GD980002 | Link 증분 추적 | Internal Loader (write) |
| TM_GD970101 | 결과 매핑 (참조) | Internal Loader (read/write) |

## Phase 2: DDL 변환 + 테이블 생성

### 대소문자 전략
- **Oracle 기본값(대문자) 사용** — 쌍따옴표 없이 생성
- 코드에서 qi() Oracle 분기: 인용 없이 반환 (Oracle이 자동 대문자 처리)
- 코드 내 소문자 테이블명/컬럼명 → Oracle에서 대소문자 무관 매칭

### PG → Oracle 타입 변환
| PostgreSQL | Oracle |
|------------|--------|
| VARCHAR(n) | VARCHAR2(n) |
| TEXT | VARCHAR2(4000) |
| SERIAL / BIGSERIAL | NUMBER + GENERATED BY DEFAULT AS IDENTITY |
| INTEGER | NUMBER(10) |
| BIGINT | NUMBER(19) |
| NUMERIC / DOUBLE | NUMBER |
| TIMESTAMP | TIMESTAMP |
| DATE | DATE |
| TIME | VARCHAR2(8) (HHmmss 문자열) |

### DDL 파일
- `scripts/oracle-init.sql` — GIMS Target 4개 테이블 ✅ 생성완료
- `scripts/oracle-init-if.sql` — IF 테이블 2개 (신규)

### IF 테이블 Oracle DDL (신규)

```sql
-- IF_RSV_SEC_JEWON: 내부 수신 제원
CREATE TABLE IF_RSV_SEC_JEWON (
    ID           NUMBER(10) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    OBSV_CODE    VARCHAR2(255) NOT NULL,
    OBSV_NAME    VARCHAR2(255),
    WELL         NUMBER(10),
    SIDO         VARCHAR2(255),
    SIGUNGU      VARCHAR2(255),
    UPMYUNDO     VARCHAR2(255),
    BUNJI        VARCHAR2(255),
    RI           VARCHAR2(255),
    X            VARCHAR2(255),
    Y            VARCHAR2(255),
    PYOGO        NUMBER,
    INSDATE      DATE,
    GULDEP       NUMBER,
    GULDIA       NUMBER,
    REGDATE      DATE,
    CASING_HEIGHT NUMBER,
    SOURCE_REFS  VARCHAR2(4000) NOT NULL,
    LINK_STATUS  VARCHAR2(20) DEFAULT 'PENDING',
    EXTRACTED_AT TIMESTAMP,
    UPDATED_AT   TIMESTAMP,
    EXECUTION_ID VARCHAR2(255),
    CONSTRAINT UK_IF_RSV_JEWON_SREFS UNIQUE (SOURCE_REFS)
);
CREATE INDEX IDX_IF_RSV_JEWON_EXEC ON IF_RSV_SEC_JEWON (EXECUTION_ID);

-- IF_RSV_SEC_OBSVDATA: 내부 수신 관측데이터
CREATE TABLE IF_RSV_SEC_OBSVDATA (
    ID           NUMBER(10) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    OBSV_CODE    VARCHAR2(255) NOT NULL,
    OBSV_DATE    DATE NOT NULL,
    OBSV_TIME    VARCHAR2(8),
    GWDEP        NUMBER,
    GWTEMP       NUMBER,
    EC           NUMBER,
    REMARK       VARCHAR2(255),
    SOURCE_REFS  VARCHAR2(4000) NOT NULL,
    LINK_STATUS  VARCHAR2(20) DEFAULT 'PENDING',
    EXTRACTED_AT TIMESTAMP,
    UPDATED_AT   TIMESTAMP,
    EXECUTION_ID VARCHAR2(255),
    CONSTRAINT UK_IF_RSV_OBSV_SREFS UNIQUE (SOURCE_REFS)
);
CREATE INDEX IDX_IF_RSV_OBSV_EXEC ON IF_RSV_SEC_OBSVDATA (EXECUTION_ID);
```

**주의**: PG에서 `obsv_time`은 `java.sql.Time` → Oracle에서는 `VARCHAR2(8)` (HHmmss 형식)
- SourceToIfStep이 소스(DMZ SND)에서 Time 타입으로 읽어서 Oracle IF에 쓸 때 문자열 변환 필요

## Phase 3: 코드 수정

### 3-1. Oracle JDBC 드라이버 추가
**파일**: `sync-agent-bojo-int/build.gradle`
```gradle
runtimeOnly 'com.oracle.database.jdbc:ojdbc11'  // JDK 17 호환
```

### 3-2. common — Multi-DB 분기 확장

#### SourceToIfStep.java
| 위치 | 현재 | Oracle 변환 |
|------|------|------------|
| `isMysql()` 옆 | 없음 | `isOracle()` 헬퍼 추가 |
| `qi()` | PG: `"name"` | Oracle: `name` (인용 없이, 자동 대문자) |
| `buildUpsertSql()` | PG: `ON CONFLICT DO UPDATE` | Oracle: `MERGE INTO ... USING (SELECT ? FROM DUAL) ON (...) WHEN MATCHED THEN UPDATE ... WHEN NOT MATCHED THEN INSERT ...` |
| `buildTimeExpression()` | PG: `(date + time)` | Oracle: time이 VARCHAR2이므로 `TO_TIMESTAMP(TO_CHAR(date, 'YYYYMMDD') \|\| ' ' \|\| time, 'YYYYMMDD HH24MISS')` |
| `updateSourceLinkStatus()` | PG: `col::text` | Oracle: `TO_CHAR(col)` |

**주의**: SourceToIfStep은 common 모듈 → DMZ bojo에서도 사용됨. `else if (isOracle)` 분기로 기존 PG/MySQL 로직에 영향 없도록 처리.

#### ConditionBuilder.java
| 위치 | 현재 | Oracle 변환 |
|------|------|------------|
| `quoteIdentifier()` | PG: `"name"` | Oracle: `name` (인용 없이) |

### 3-3. bojo-int — GimsTargetRepository Oracle 호환
**파일**: `GimsTargetRepository.java`

현재 PG 전용 SQL → Oracle 호환으로 변환. **dbType 파라미터 추가** 필요.

| 메서드 | 현재 (PG) | Oracle 변환 |
|--------|----------|------------|
| `ensureResultId()` | `INSERT ... ON CONFLICT DO NOTHING RETURNING result_id` | SELECT 먼저 → 없으면 INSERT → SELECT |
| `batchUpsertLink()` | `INSERT ... ON CONFLICT (pk) DO UPDATE SET col=EXCLUDED.col` | `MERGE INTO ... USING (SELECT ?,?,... FROM DUAL) src ON (...) WHEN MATCHED THEN UPDATE SET ... WHEN NOT MATCHED THEN INSERT ...` |

### 3-4. bojo-int — InternalBojoLoadStep
- GimsTargetRepository에 dbType 전달 필요
- DataSourceProvider.getDbType(targetDsId)로 가져옴

### 3-5. Datasource 설정 변경
- Orchestrator UI에서 `internal` datasource를 Oracle로 변경
  - `dbType: "ORACLE"`, `host: localhost`, `port: 29004`, `database: XEPDB1`
- source + target 모두 같은 Oracle datasource 사용

### 3-6. Agent 자체 DB는 PG 유지
- `application.yml`의 `spring.datasource` / `hibernate.dialect` → **변경 없음** (PG 유지)
- SyncLog 등 JPA 엔티티는 Agent 자체 PG DB에 저장
- IF/Target은 동적 JdbcTemplate 접근 → Oracle

## Phase 4: E2E 검증
1. Oracle 컨테이너 기동 + DDL 실행 (target + IF)
2. Orchestrator에 Oracle datasource 등록 (internal → Oracle)
3. Internal RCV 파이프라인 실행 → IF_RSV 테이블 적재 확인
4. Internal Loader 파이프라인 실행 → GIMS Target 적재 확인
5. SyncLog 정상 기록 확인 (PG)
6. Retention cleanup Oracle 동작 확인

## 영향 범위
- **bojo-int**: GimsTargetRepository + InternalBojoLoadStep 수정
- **common**: SourceToIfStep + ConditionBuilder에 Oracle 분기 추가 (기존 PG/MySQL 영향 없음)
- **bojo (DMZ)**: common 변경 시 빌드 확인 필요. DMZ는 Oracle 미사용 → 분기 타지 않음
- **Orchestrator/Proxy**: 코드 변경 없음, datasource 등록만

## 리스크
- IDENTITY 컬럼 동작이 PG와 미세하게 다를 수 있음 (12c+ 지원)
- `java.sql.Time` → Oracle VARCHAR2 변환 시 SourceToIfStep 데이터 타입 핸들링 주의
- 대소문자: Oracle 인용 없으면 자동 대문자이므로 코드 내 소문자 테이블명도 동작하지만, 메타데이터 조회 시 대문자로 반환됨
- common 수정 시 DMZ Agent에 side-effect 없는지 빌드로 확인
