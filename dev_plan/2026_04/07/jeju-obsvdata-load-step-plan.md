# I2: JejuObsvdataLoadStep 구현 계획

> 작성일: 2026-04-07
> 선행: I1(JejuJewonLoadStep) 완료, IF_RSV_TB_JEJU 데이터 존재

## 목적

제주 관측 데이터(수위/수온/전기전도도)를 IF 테이블에서 GIMS 타겟 테이블로 적재한다.

## 소스 → 타겟

```
IF_RSV_TB_JEJU (IF 테이블)
  │
  ├─ msn = "S11" ──→ PM_GD970201 (ODM관측자료, 단일심도)
  │                    × 3건 (GL, WTEMP, SCOND)
  │
  └─ msn = "S2X" ──→ PM_GD970202 (ODM다심도관측자료)
                       × 3건 (GL, WTEMP, SCOND)
                       PSTN_SN = X (심도 위치)
```

## IF 소스 컬럼 (IF_RSV_TB_JEJU)

| 컬럼 | 용도 |
|------|------|
| ID | PK (auto-increment) |
| OBSRVT_ID | 관측소 식별자 → TM_GD970001.OBSRVT_ID → BRNCH_ID 확보 |
| YMD | 날짜 (YYYYMMDD) |
| DATA_TIME | 시각 (HH24MISS) |
| GL | 수위 값 |
| WTEMP | 수온 값 |
| SCOND | 전기전도도 값 |
| MSN | 센서 구분 (S11=단일심도, S21/S22/...=다심도) |
| SOURCE_REFS | 추적용 |
| LINK_STATUS | PENDING → SUCCESS/FAILED |

## 타겟 테이블

| 컬럼 | 타입 | NN | 값 |

| OBSRVN_DATA_ID | NUMBER(10) IDENTITY | Y | 자동생성 |
| RSLT_ID | NUMBER(10) | Y | TM_GD970101에서 조회 (BRNCH_ID + OBSRVN_ARTCL_ID) |
| OBSRVN_DATA_VL | VARCHAR2(20) | | GL/WTEMP/SCOND 값 |
| OBSRVN_DT | DATE | Y | YMD + DATA_TIME → TO_DATE |

| EXECUTION_ID | VARCHAR2(255) | | 추적용 |
| SOURCE_REFS | VARCHAR2(4000) | | IF 테이블 source_refs |

### PM_GD970202 (ODM다심도관측자료 - 다심도, msn=S2X)

| 컬럼 | 타입 | NN | 값 |

| SOURCE_REFS | VARCHAR2(4000) | | IF 테이블 source_refs |

## 핵심 로직

### 1. RSLT_ID 조회

```sql
SELECT RSLT_ID FROM TM_GD970101
WHERE BRNCH_ID = (SELECT BRNCH_ID FROM TM_GD970001 WHERE OBSRVT_ID = ?)
  AND OBSRVN_ARTCL_ID = ?
```

- OBSRVN_ARTCL_ID: GL=5, WTEMP=163, SCOND=52

### 2. 날짜 변환

```
YMD(20260407) + DATA_TIME(143000) → TO_DATE('20260407143000', 'YYYYMMDDHH24MISS')
```

### 3. MSN 분기

```
S11 → PM_GD970201 (PSTN_SN 없음)
S21 → PM_GD970202 (PSTN_SN = 1)
S22 → PM_GD970202 (PSTN_SN = 2)
S2N → PM_GD970202 (PSTN_SN = N)
```

### 4. 행 당 처리

IF 1건 → 타겟 최대 3건 (GL, WTEMP, SCOND 각각 INSERT)
- GL 값이 null/empty면 GL INSERT 건너뜀 (WTEMP, SCOND 동일)

## DDL (PM_GD970202 신규 생성 필요)

PM_GD970201은 이미 존재 (scripts/oracle-standardize-bojo-target.sql).
PM_GD970202는 **신규 DDL 필요**.

```sql
CREATE TABLE PM_GD970202 (
    OBSRVN_DATA_ID  NUMBER(10)    GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    RSLT_ID         NUMBER(10)    NOT NULL,
    PSTN_SN         NUMBER(22)    NOT NULL,
    OBSRVN_DATA_VL  VARCHAR2(20),
    OBSRVN_DT       DATE          NOT NULL,
    QLT_ID          NUMBER(10)    NOT NULL,
    EXECUTION_ID    VARCHAR2(255),
    SOURCE_REFS     VARCHAR2(4000),
    CONSTRAINT PK_PM_GD970202 PRIMARY KEY (OBSRVN_DATA_ID)
);

COMMENT ON TABLE PM_GD970202 IS 'ODM다심도관측자료';
COMMENT ON COLUMN PM_GD970202.OBSRVN_DATA_ID IS '관측자료아이디';
COMMENT ON COLUMN PM_GD970202.RSLT_ID IS '결과아이디';
COMMENT ON COLUMN PM_GD970202.PSTN_SN IS '위치순번';
COMMENT ON COLUMN PM_GD970202.OBSRVN_DATA_VL IS '관측자료값';
COMMENT ON COLUMN PM_GD970202.OBSRVN_DT IS '관측일시';
COMMENT ON COLUMN PM_GD970202.QLT_ID IS '품질아이디';

CREATE INDEX IDX_PM_GD970202_RSLT_ID ON PM_GD970202(RSLT_ID);
CREATE INDEX IDX_PM_GD970202_OBSRVN_DT ON PM_GD970202(OBSRVN_DT);
CREATE INDEX IDX_PM_GD970202_EID ON PM_GD970202(EXECUTION_ID);
```

## 수정 대상 파일

| 파일 | 변경 | 비고 |
|------|------|------|
| `scripts/oracle-init-jeju-target.sql` | PM_GD970202 DDL 추가 | 신규 테이블 |
| `JejuLoadStepFactory.java` | `jeju-obsvdata-load` factory-key 추가 | 기존 파일 수정 |
| `JejuObsvdataLoadStep.java` | **신규** — I2 Step 구현 | I1 패턴 참조 |
| `internal-jeju-loader.yml` | step 추가 (jeju-obsvdata-load) | 기존 YAML 수정 |

## YAML 추가 내용

```yaml
steps:
  - id: jeju-jewon-load        # 기존 I1
    ...
  - id: jeju-obsvdata-load      # 신규 I2
    name: 제주 관측데이터 적재
    factory-key: jeju-obsvdata-load
    source-table: IF_RSV_TB_JEJU
    target-table: [PM_GD970201, PM_GD970202]
```

## 구현 순서

```
1. PM_GD970202 DDL 생성 (Oracle 29004)
2. JejuObsvdataLoadStep.java 구현
3. JejuLoadStepFactory에 factory-key 추가
4. internal-jeju-loader.yml에 step 추가
5. 빌드 + 기동
6. E2E 테스트 (IF_RSV_TB_JEJU 데이터 → PM_GD970201/PM_GD970202)
7. auto-discover로 Agent 재등록 테스트 (선택)
```

## 주의사항

- RSLT_ID가 없으면 (제원 미등록) 해당 건 SKIP + 경고 로그
- 값이 빈 문자열이면 INSERT 건너뜀 (GL/WTEMP/SCOND 각각 독립)
- MERGE가 아닌 INSERT — 관측 데이터는 시계열이므로 중복키 시 SKIP (ON CONFLICT 무시)
- I1(제원)이 먼저 실행되어야 BRNCH_ID/RSLT_ID 참조 가능
