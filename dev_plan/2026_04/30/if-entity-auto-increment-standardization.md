## 1. 배경

### 발견 경로 (4/30)
- 약수터 SND 첫 실행 → 1000/1000 fail (`null value in column "sn" violates not-null constraint`)
- 원인 추적: `SourceToTargetStep` default `["id","sn"]` 가 sn INSERT 제외 → target sequence 없음 → null 위반
- 사용자 의문 ("같은 패턴인데 왜 약수터만 깨지나") 검증 → 모듈별 entity 정의 일관성 깨진 사실 발견

### 검증된 사실 (모듈별 IF_SND/IF_RSV 비교)

| 모듈 / 패턴 | entity `@GeneratedValue` 명시 | DB sequence/IDENTITY | 동작 |
|---|:-:|:-:|:-:|
| bojo `if_snd_sec_*`, `if_rsv_sec_*` (Integer id PK) | ❌ | ✅ (Hibernate Integer auto-detect 추정) | 운 좋게 동작 |
| jeju `if_snd_use_jeju_day` (Long id PK) | ✅ IDENTITY | ✅ | 정합 (정석) |
| **others `if_snd_*` (Long sn PK) — 6 테이블** | **❌** | **❌** | **❌ 깨짐** |
| Oracle bojo-int `IF_RSV_*.ID` | ✅ IDENTITY | ✅ | 정합 |
| Oracle bojo-int `IF_RSV_*.SN` (일반 컬럼, 일부 테이블) | (PK 아님) | nullable Y | silent 손실 (별개 트랙) |

### 옛 데이터 검증
| 테이블 | 적재 시점 | sn 값 |
|---|---|---|
| if_snd_sec_jewon (id PK) | 4/02 | id sequence 자동채번 (source.id 와 다름) |
| if_snd_tm_gd014001 (sn PK) | 4/03 | source.sn 그대로 (4/23 변경 이전, sn INSERT 포함) |
| if_snd_use_legacy_data (sn PK) | 4/03 | source.sn 그대로 |
| IF_RSV_TM_GD014001 (Oracle, SN 일반) | 4/14 | source.SN 그대로 (4/23 이전) |

→ **4/23 default 변경 이후 새로 SND 돌린 Agent = 약수터뿐**. 다른 sn PK IF_SND 도 새로 돌리면 동일 fail 가능성.

## 2. 룰 재정비 (사용자 결정)

> **표준 룰**:
> 1. **Target IF entity 의 PK 는 자동채번 명시 (`@GeneratedValue(strategy = GenerationType.IDENTITY)`) 가 기본**
>    - `IF_SND_*`, `IF_RSV_*`, `Target Table` 모두 의무
>    - Source 의 PK 값은 `source_refs` 메타로 추적 (별도 컬럼에 보존 X)
> 2. **Source 와 Target 이 겹치는 이름의 비-PK 비즈니스 컬럼이 있을 때만 YAML `exclude-insert-columns` 로 조정**
>    - 예: A7 (`@Id id IDENTITY` + `sn` 비즈니스값) → `exclude-insert-columns: ["id"]`
> 3. `default ["id","sn"]` = "PK 가 id 또는 sn 이름이면 자동채번" 규약. (1) 정합 시 자연 동작.

### Step 별 적용 위치
| Step | INSERT 컬럼 결정 | YAML 조정 |
|---|---|---|
| `SourceToTargetStep` (SND/RCV) | YAML `exclude-insert-columns` + default | ✅ |
| `SimpleLoadStep` (Loader) | 코드 내 하드코딩 (`!SN.equalsIgnoreCase(c)`) | ❌ (코드 책임) |
| 커스텀 Step (Loader) | 자체 매핑 | ❌ |

→ **모든 Step 의 공통 전제 = "Target entity 의 PK 가 자동채번"**.

**엔티티 수정 패턴** (jeju 표준 따라):
```java
@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
@Column(name = "sn")
@Comment("일련번호 (PK)")
private Long sn;
```

**DB 마이그 SQL 패턴** (PG):
```sql
-- 6 테이블 각각 (max(sn)+1 또는 1 시작)
ALTER TABLE if_snd_tm_gd010310 ALTER COLUMN sn ADD GENERATED BY DEFAULT AS IDENTITY;
SELECT setval(pg_get_serial_sequence('if_snd_tm_gd010310', 'sn'),
              GREATEST(COALESCE((SELECT MAX(sn) FROM if_snd_tm_gd010310), 0) + 1, 1));
```
